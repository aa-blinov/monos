# Система организации заметок (Monos)

Данный репозиторий организован по принципу тематического разделения сфер жизни и работы для удобной навигации и быстрого поиска информации.

## Структура каталогов

- `Работа/` — Все, что связано с профессиональной деятельностью.
    - `EORA/` — Проекты, стратегии и материалы по работе в EORA.
    - `NSTU/` — Коммерческие предложения и взаимодействие с заказчиками.
    - `Исследования DPI/` — Материалы по анализу пакетов.
- `Проекты и Образование/` — Учеба и личные проекты.
    - `Проект ВШЭ/` — Проект "Стивен" для ВШЭ.
    - `Образование жены/` — Учебные материалы.
    - `Личные проекты/` — Собственные технические разработки.
- `Образование/` — База знаний по технологиям и программированию.
    - `Большие языковые модели/` — LLM, RAG, prompt engineering и AI.
    - `Питон/` — Особенности языка и примеры кода.
    - `Копилот/` — Промпты и кейсы использования ИИ-помощников.
- `Документы/` — Релокация и официальные документы.
    - `Цифровой кочевник/` — Подача на Digital Nomad Residency (CV, мотивационные письма).
    - `ВНЖ/` — Вопросы ВНЖ, РПП и трактовки законов.
- `Личное и Творчество/` — Личное развитие и творчество.
    - `Психология/` — Дневники (эмпатия), заметки по психологии.
    - `Идеи/` — Сборник идей для будущих проектов.
    - `Промпты/` — Избранные промпты для работы с ИИ.
- `assets/` — Статические файлы.
    - `images/` — Скриншоты и изображения.
    - `docs/` — Внешние документы (docx, pdf).

## Правила распределения

1. **Работа vs Проекты:** Если деятельность приносит прямой доход или связана с текущим работодателем (EORA, NSTU), она идет в `Работа/`. Если это самообучение или неоплачиваемый проект — в `Проекты и Образование/`.
2. **Образование:** В `Образование/` кладем только "чистые" знания (гайды, теорию), а не реализацию конкретного рабочего проекта.
3. **Медиафайлы:** Все картинки и тяжелые документы должны находиться в соответствующих папках внутри `assets/`, а в заметках использоваться ссылки на них.
4. **Именование:** Используйте пробелы в названиях папок (вместо подчеркиваний) для удобства, но все должно правильно работать благодаря кросс-платформенной поддержке.

## Первичная настройка (после клонирования)

### Предварительные требования

Убедитесь, что установлены:
- ✅ `uv` — менеджер проектов (https://docs.astral.sh/uv/getting-started/)
- ✅ `git` — контроль версий
- ✅ Python 3.12+

### Инициализация окружения

Выберите команду в зависимости от вашей ОС:

**Linux / macOS:**
```bash
python3 setup.py
```

**Windows:**
```cmd
python setup.py
```
или
```cmd
setup.bat
```

Скрипт автоматически:
1. Проверит наличие `uv` и `git`
2. Установит все зависимости через `uv`
3. Установит и настроит `pre-commit` hooks
4. Подготовит окружение к работе

### После инициализации

При каждом `git commit` будут автоматически выполняться проверки:
- 📝 **mdformat** — Форматирование Markdown файлов (wrap 120 символов)
- 🐍 **ruff** — Линтинг и форматирование Python кода
- ✓ **pre-commit-hooks** — Базовые проверки (окончания файлов, YAML, JSON, большие файлы)

Если нужно пропустить проверки (не рекомендуется):
```bash
git commit --no-verify
```

Для проверки всех файлов вручную:
```bash
pre-commit run --all-files
```

## Автоматизация

### ⚡ Быстрый запуск через Make (рекомендуется)

Используйте `make` для запуска скриптов без зависания `uv`:

```bash
# Показать все доступные команды
make help

# Создать новую заметку
make new-note

# Отформатировать все Markdown файлы
make format-notes

# Проверить мертвые ссылки
make check-links

# Добавить фронтматтер ко всем файлам
make add-frontmatter

# Установить зависимости
make install
```

### Запуск скриптов напрямую через Python

Если `make` недоступен, запускайте скрипты напрямую:

```bash
# Создать новую заметку
python3 .scripts/new_note.py

# Отформатировать все заметки
python3 .scripts/format_notes.py

# Проверить мертвые ссылки
python3 .scripts/check_links.py
python3 .scripts/check_links.py --verbose
python3 .scripts/check_links.py -v

# Добавить фронтматтер ко всем файлам
python3 .scripts/add_frontmatter.py
```

⚠️ **НЕ рекомендуется** использовать `uv run .scripts/...` так как `uv` может зависнуть при инициализации проекта. Используйте либо `make`, либо `python3` напрямую.

### Встроенные инструменты

Все используемые инструменты настроены в `pyproject.toml` и `.pre-commit-config.yaml`:

- **mdformat** — Форматирование Markdown с поддержкой GFM и frontmatter
- **ruff** — Быстрый Python линтер и форматер
- **pre-commit** — Автоматизация проверок при коммитах
