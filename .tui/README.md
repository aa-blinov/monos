# Zed Notes TUI — Управление заметками через терминал

Интерактивное терминальное приложение для управления вашей базой знаний на базе Textual.

## Возможности

### Навигация и просмотр
- 📁 **Браузер заметок** — иерархическая структура папок с файлами
- 📋 **Превью** — показ превью содержимого файла на правой панели
- 🔍 **Поиск** — поиск по названиям файлов и содержимому
- ⏰ **Недавние** — быстрый доступ к 10 последним измененным заметкам
- ℹ️ **Информация** — детальные метаданные файла (размер, дата, теги и т.д.)

### Редактирование и управление файлами
- ✏️ **Редактирование** — открытие файла в вашем редакторе ($EDITOR)
- 📄 **Открытие** — открытие файла в системном приложении
- 🗂️ **Переименование** — переименование файлов и папок
- 🗑️ **Удаление** — удаление с подтверждением
- 🏷️ **Теги** — просмотр метаданных и тегов заметки

### Создание и форматирование
- ✨ **Новая заметка** — интерактивное создание заметки с категорией, названием и тегами
- 🎨 **Форматирование** — автоматическое форматирование всех заметок (mdformat)
- 📝 **YAML фронтматтер** — автоматическое добавление метаданных

### Git интеграция
- 🔄 **Синхронизация** — автоматический коммит и пуш изменений
- 📦 **История** — все операции можно откатить через Git

## Как запустить

### Через Make (рекомендуется)
```bash
make notes-ui
```

### Напрямую через Python
```bash
uv sync --group tui
uv run --group tui .tui/cli.py
```

### Без uv (если uv недоступен)
```bash
python3 .tui/cli.py
```

## Горячие клавиши

### Основные команды
| Клавиша | Действие |
|---------|----------|
| `q` | Выход из приложения |
| `?` | Справка (помощь) |
| `Escape` | Закрыть диалог/вернуться |

### Создание и поиск
| Клавиша | Действие |
|---------|----------|
| `n` / `Ctrl+N` | Создать новую заметку |
| `/` | Открыть поиск |
| `Ctrl+R` | Показать недавние заметки |

### Редактирование файлов
| Клавиша | Действие |
|---------|----------|
| `e` | Редактировать файл в $EDITOR (nano/vim/nvim) |
| `o` | Открыть файл в системном приложении |
| `r` | Переименовать файл |
| `d` | Удалить файл (с подтверждением) |
| `t` | Показать теги и метаданные |
| `i` | Открыть детальную информацию |

### Сохранение и синхронизация
| Клавиша | Действие |
|---------|----------|
| `Ctrl+S` | Синхронизировать с Git (add, commit, push) |
| `Ctrl+F` | Форматировать все заметки (mdformat) |

### Навигация в дереве
| Клавиша | Действие |
|---------|----------|
| `↑` / `↓` или `j` / `k` | Движение вверх/вниз |
| `←` / `→` или `h` / `l` | Сворачивание/раскрытие папок |
| `Enter` / `Space` | Выбрать файл |

## Работа с заметками

### Создание новой заметки

1. Нажмите `n` (или `Ctrl+N`)
2. Введите категорию (например: `Образование/Большие языковые модели`)
3. Введите название заметки
4. Добавьте теги (через запятую, опционально)
5. Нажмите кнопку "Создать" или `Ctrl+S`

Система автоматически:
- Создаст файл с нижними подчеркиваниями вместо пробелов
- Добавит YAML фронтматтер с метаданными
- Разместит файл в правильную папку

### Редактирование файла

1. Выберите файл в браузере (кликните или используйте стрелки)
2. Нажмите `e` для редактирования
3. Отредактируйте содержимое в вашем редакторе
4. Сохраните файл и закройте редактор
5. TUI перезагрузит список заметок

**Совет:** Установите переменную окружения `EDITOR`:
```bash
export EDITOR=vim      # или nano, nvim и т.д.
```

### Управление файлами

#### Переименование
1. Выберите файл
2. Нажмите `r`
3. Введите новое имя
4. Подтвердите

#### Удаление
1. Выберите файл
2. Нажмите `d`
3. Подтвердите удаление (нажмите `y` или кнопку "Да")

#### Просмотр информации
1. Выберите файл
2. Нажмите `i`
3. Увидите полную информацию о файле:
   - Путь и размер
   - Дата создания и изменения
   - YAML метаданные (название, статус, категория, теги)

## Синхронизация с Git

### Автоматическая синхронизация
```
Ctrl+S
```

Выполнит:
1. `git add -A` — добавит все изменения
2. `git commit -m "Auto-sync from TUI"` — создаст коммит
3. `git push` — отправит изменения на удаленный сервер

### Форматирование перед синхронизацией
```
Ctrl+F
```

Выполнит:
- Форматирование всех `.md` файлов
- Нормализацию ссылок
- Проверку YAML фронтматтера

## Архитектура TUI

```
.tui/
├── cli.py           # Точка входа CLI
├── app.py           # Главное приложение и все экраны
├── models.py        # Бизнес-логика (NotesManager)
├── widgets.py       # Пользовательские виджеты Textual
└── README.md        # Документация (этот файл)
```

### Классы

#### `NotesManager`
Управление файлами и директориями:
- Получение списка заметок
- Поиск по файлам
- Создание новых заметок
- Работа с YAML метаданными
- Git операции

#### `MainScreen`
Главный экран приложения с:
- Браузером файлов (слева)
- Панелью информации (справа)
- Статус-баром
- Футером с горячими клавишами

#### `NotesBrowser`
Виджет для отображения иерархического дерева файлов

#### Диалоговые экраны
- `NewNoteScreen` — создание заметки
- `SearchScreen` — поиск по файлам
- `RenameScreen` — переименование
- `InfoScreen` — детальная информация
- `HelpScreen` — справка
- `ConfirmScreen` — подтверждение действий

## Расширение функциональности

### Добавление новых команд

1. Добавьте метод `action_XXX()` в `MainScreen`
2. Добавьте клавишу в `BINDINGS`
3. Реализуйте логику действия

Пример:
```python
def action_my_command(self) -> None:
    """Мое пользовательское действие"""
    self.update_status("Выполняю команду...")
    # Логика здесь
    self.update_status("✓ Готово")
```

### Добавление новых экранов

1. Создайте класс, наследующий `Screen`
2. Реализуйте `compose()` для UI
3. Добавьте обработчики событий
4. Используйте `self.app.push_screen()` для открытия

## Известные ограничения

- На очень больших директориях (5000+ файлов) может быть медленнее
- Некоторые возможности $EDITOR (как интерактивный режим) могут не работать

## Советы и трюки
