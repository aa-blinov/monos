# Инструкции для AI-агентов (Zed Notes)

Этот файл содержит правила и контекст для AI-агентов, работающих с данной базой знаний. При взаимодействии с репозиторием агент должен придерживаться следующих принципов.

## Роли и обязанности

### 1. Архитектор (Structure Manager)
- **Задача:** Поддерживать чистоту файловой структуры и единообразие формата.
- **Действия:** 
    - Следить, чтобы новые заметки не создавались в корне.
    - Перемещать файлы согласно правилам в `README.md`.
    - Использовать `_` вместо пробелов в названиях папок.
    - **Форматирование:** После внесения изменений или добавления новых файлов в базу знаний, агент должен запустить скрипт форматирования командой `uv run format_notes.py`.
    - **Создание заметок:** Для создания новых заметок использовать интерактивный скрипт `uv run new_note.py`, который автоматически поместит файл в правильную категорию.

### 2. Редактор (Knowledge Curator)
- **Задача:** Оптимизация содержания заметок.
- **Действия:**
    - При чтении длинных файлов (например, исследования DPI) выделять ключевые тезисы.
    - Добавлять перекрестные ссылки между `Работа/` и `Технические_Знания/` (например, связь рабочего проекта RAG с базой знаний по ЯМЗ).
    - Проверять наличие ссылок на изображения в `assets/images/`.

### 3. Контекстный ассистент (Context Specialist)
- **Задача:** Помогать пользователю находить информацию по ключевым темам.
- **Контекст:**
    - **Работа/EORA, Работа/NSTU:** Коммерческая разработка, RAG, бенчмарки, пресейлы.
    - **Документы:** Релокация, Казахстан, Digital Nomad, ВНЖ (РПП).
    - **Технические_Знания:** Python, LLM, DPI, Scapy.

## Технические правила работы

### Скрипты автоматизации

#### 1. Форматирование заметок
- **Скрипт:** `format_notes.py` в корне проекта.
- **Что делает:** Проверяет все `.md` файлы и приводит их к единому стандарту (GFM, без жесткого переноса строк).
- **Команда:** `uv run format_notes.py`
- **Когда запускать:** После каждого внесения изменений перед коммитом.

#### 2. Создание новой заметки
- **Скрипт:** `new_note.py` в корне проекта.
- **Что делает:**
  - Отображает интерактивное древовидное меню со всеми категориями.
  - Запрашивает выбор основной категории (Работа, Проекты_и_Образование, Технические_Знания и т.д.).
  - При выборе показывает подкатегории (EORA, NSTU, ЯМЗ и т.д.) с визуальными маркерами.
  - Позволяет создать новую подкатегорию прямо во время навигации.
  - Позволяет вернуться на шаг назад (введи `0`).
  - Запрашивает название заметки и опциональные теги.
  - Автоматически генерирует имя файла (lowercase, с подчеркиваниями вместо пробелов).
  - Создает файл с YAML фронтматтером (title, date, category, tags).
  - Предлагает открыть файл в редакторе.
- **Команда:** `uv run new_note.py`
- **Когда использовать:** Вместо ручного создания файлов в корне или в неправильных местах.

### Остальные правила

1. **Форматирование путей:** Всегда использовать относительные пути.
2. **Изображения:** Если в тексте встречается `Pasted image...`, агент должен убедиться, что файл находится в `assets/images/`.
3. **Markdown:** Использовать заголовки первого уровня (#) только один раз в начале файла.
4. **Кодировка:** Сохранять файлы строго в UTF-8.

## Алгоритм сортировки новых данных (Inbox Zero)

При обнаружении новых файлов в корне:
1. Проанализировать содержимое.
2. Если файл относится к работе (EORA/NSTU/Research) -> `Работа/`.
3. Если это гайд или код без привязки к проекту -> `Технические_Знания/`.
4. Личные размышления или психология -> `Личное_и_Творчество/`.
5. Релокация и документы -> `Документы/`.
6. Скриншоты -> `assets/images/`.

## Git и версионирование

- После создания или изменения заметок: запустить `uv run format_notes.py`.
- Регулярно делать коммиты с описательными сообщениями.
- Перед push убедиться, что все скрипты пройдены без ошибок.
