# Скрипты автоматизации Zed Notes

Все скрипты находятся в директории `.scripts/` и автоматизируют работу с заметками.

## Как запускать скрипты

### ⚡ Способ 1: Make (рекомендуется)

```bash
make help              # Показать все команды
make new-note          # Создать новую заметку
make format-notes      # Форматировать все файлы
make check-links       # Проверить ссылки
make add-frontmatter   # Добавить фронтматтер
```

### Способ 2: Python напрямую

```bash
python3 .scripts/new_note.py
python3 .scripts/format_notes.py
python3 .scripts/check_links.py
python3 .scripts/add_frontmatter.py
```

### ❌ НЕ рекомендуется: uv run

```bash
# uv run может зависнуть при инициализации проекта!
uv run .scripts/new_note.py   # Может зависнуть
```

## Описание скриптов

### `new_note.py`
Интерактивное создание новой заметки с YAML фронтматтером.

**Что делает:**
- Предлагает выбрать категорию из существующих или создать новую
- Запрашивает название заметки
- Запрашивает теги (опционально)
- Создает файл с правильным фронтматтером

**Используется в:** каждый день при создании заметки

---

### `format_notes.py`
Форматирует все Markdown файлы по стандартам проекта.

**Что делает:**
- Применяет mdformat ко всем `.md` файлам
- Оборачивает текст на 120 символов
- Форматирует YAML фронтматтер
- Нумерует списки

**Используется:** перед коммитом (автоматически через pre-commit)

---

### `check_links.py`
Проверяет все ссылки между заметками на наличие мертвых ссылок.

**Что делает:**
- Находит все относительные ссылки в `.md` файлах
- Проверяет, существуют ли файлы
- Выводит список ошибок

**Опции:**
- `--verbose` или `-v` — подробный вывод с информацией о каждой ссылке

**Используется:** перед публикацией или после переноса файлов

---

### `add_frontmatter.py`
Добавляет YAML фронтматтер к файлам, которые его не имеют.

**Что делает:**
- Сканирует все `.md` файлы в проекте
- Для файлов без фронтматтера генерирует его:
  - `title` — из имени файла
  - `date` — текущая дата в ISO 8601
  - `category` — из пути в структуре
  - `tags` — пустой массив
  - `status` — `draft` по умолчанию
- Пропускает файлы в `docs/` (системная документация)
- Пропускает файлы, которые уже имеют фронтматтер

**Используется:** один раз при унификации структуры проекта

---

## Проблемы

### uv зависает при запуске скриптов

**Причина:** `uv run` пытается инициализировать проект перед запуском.

**Решение:** используйте `make` или `python3` напрямую.

### Python не найден

```bash
# Убедитесь, что Python 3.12+ установлен
python3 --version

# Если не установлен:
# macOS: brew install python@3.12
# Linux: sudo apt install python3.12
# Windows: https://www.python.org/downloads/
```

### Make не установлен

```bash
# macOS
brew install make

# Linux (Ubuntu/Debian)
sudo apt install make

# Windows (использовать WSL2 или Git Bash)
```

## Интеграция с Git

Все скрипты интегрированы с `pre-commit` hooks:
- При `git commit` автоматически запускается форматирование
- Можно запустить вручную: `pre-commit run --all-files`

## Расширение

Если нужно добавить новый скрипт:

1. Создайте `new_script.py` в `.scripts/`
2. Добавьте функцию `main()` в конец файла
3. Добавьте команду в `Makefile`
4. Обновите эту документацию

Каждый скрипт должен:
- Быть независимым (без внешних зависимостей кроме стандартной библиотеки)
- Иметь docstring в начале файла
- Выполнять одну задачу хорошо
- Быть кроссплатформенным (Windows, macOS, Linux)
