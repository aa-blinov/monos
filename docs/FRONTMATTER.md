# YAML Frontmatter Standard

## Описание

Все заметки в проекте используют унифицированный YAML фронтматтер для метаданных. Это обеспечивает:

- 📋 Единообразную структуру всех заметок
- 🔍 Возможность поиска и фильтрации по метаданным
- 📊 Автоматическую генерацию индексов и таблиц содержания
- 🔗 Улучшенную навигацию между заметками

## Обязательный формат

```yaml
---
title: Название заметки
date: 2024-05-02T15:30:00+00:00
category: Работа/EORA
tags: [ai, rag, benchmark]
status: draft
author: Александр Блинов
---
```

## Поля фронтматтера

### `title` (обязательное)

**Тип:** строка  
**Описание:** Человекочитаемое название заметки  
**Пример:** `title: "Второе место в RAG-бенчмарке RRNCB"`  
**Рекомендации:**
- Делайте название кратким (до 80 символов)
- Избегайте дублирования с названием файла
- Используйте кириллицу свободно

### `date` (обязательное)

**Тип:** ISO 8601 datetime  
**Описание:** Дата и время создания заметки  
**Пример:** `date: 2024-05-02T15:30:00+00:00`  
**Рекомендации:**
- Автоматически генерируется скриптом `new_note.py`
- Используйте ISO 8601 формат для совместимости
- UTC временная зона рекомендуется (`+00:00`)

### `category` (обязательное)

**Тип:** строка (путь)  
**Описание:** Категория заметки (соответствует структуре директорий)  
**Пример:** `category: Работа/EORA/Проекты`  
**Рекомендации:**
- Должна совпадать с расположением файла на диске
- Используйте прямые слэши `/` как разделители
- Следуйте иерархии: `Основная_Категория/Подкатегория/...`

### `tags` (обязательное)

**Тип:** список (YAML array) или пустой список  
**Описание:** Теги для категоризации и поиска  
**Примеры:**
```yaml
tags: [ai, machine-learning, rag]
tags: [python, tutorial]
tags: []  # Если нет тегов
```

**Рекомендации:**
- Используйте англ. слова в нижнем регистре
- Разделяйте слова дефисом (`machine-learning` не `machine_learning`)
- Старайтесь использовать существующие теги (для консистентности)
- 3-5 тегов в среднем — оптимально
- Если категория уже определяет тему, теги могут быть пустыми

### `status` (опциональное)

**Тип:** строка  
**Описание:** Статус разработки заметки  
**Возможные значения:**
- `draft` — черновик, незаконченная заметка
- `wip` — work in progress (в процессе редактирования)
- `published` — готовая к использованию
- `archived` — архивирована, не обновляется

**Пример:** `status: draft`  
**По умолчанию:** `draft`

### `author` (опциональное)

**Тип:** строка  
**Описание:** Автор заметки  
**Пример:** `author: Иван Блинов`  
**Рекомендации:**
- Используйте полное имя
- Для заметок коллектива перечислите авторов через запятую
- По умолчанию: основной владелец репо

### `source` (опциональное)

**Тип:** URL или строка  
**Описание:** Источник информации или связанный ресурс  
**Примеры:**
```yaml
source: https://fractalagents.ai/rrncb-rag-benchmark/
source: Книга "Искусственный интеллект" - Norvig & Russell
```

### `related` (опциональное)

**Тип:** список  
**Описание:** Ссылки на связанные заметки (относительные пути)  
**Пример:**
```yaml
related:
  - ../EORA/Проекты/KODE/EORA_KODE_Plan.md
  - ../../Технические_Знания/ЯМЗ/RAG.md
```

**Рекомендации:**
- Используйте относительные пути от текущей заметки
- Указывайте только наиболее релевантные связи (до 5)

## Полный пример фронтматтера

```yaml
---
title: Второе место в RAG-бенчмарке RRNCB
date: 2024-05-02T10:00:00+00:00
category: Работа/EORA/Проекты
tags: [rag, ai, benchmark, russian]
status: published
author: Иван Блинов
source: https://fractalagents.ai/rrncb-rag-benchmark/
related:
  - ../Проекты/Grotem/RAG_first_agreements.md
  - ../../Технические_Знания/ЯМЗ/RAG.md
---

## Второе место при вычислительных затратах в разы ниже лидера

Авандок.ИИ Ассистент занял 2-е место по итоговому рейтингу...
```

## Как использовать в Markdown

### При создании через скрипт

Просто запустите `uv run .scripts/new_note.py` и скрипт автоматически создаст правильный фронтматтер:

```bash
uv run .scripts/new_note.py
```

### При ручном создании

Скопируйте этот шаблон:

```yaml
---
title: Название вашей заметки
date: 2024-05-02T15:30:00+00:00
category: Основная_Категория/Подкатегория
tags: [tag1, tag2, tag3]
status: draft
author: Ваше имя
---

# Название вашей заметки

Содержимое заметки начинается здесь...
```

## Validation Rules

| Поле | Обязательное | Тип | Min | Max |
|------|-------------|-----|-----|-----|
| `title` | ✅ | String | 3 | 255 |
| `date` | ✅ | ISO 8601 | - | - |
| `category` | ✅ | String | 3 | 100 |
| `tags` | ✅ | Array | 0 | 10 |
| `status` | ❌ | Enum | - | - |
| `author` | ❌ | String | 2 | 100 |
| `source` | ❌ | URL/String | - | 500 |
| `related` | ❌ | Array | 0 | 10 |

## Советы и Best Practices

### ✅ Делайте так:

```yaml
---
title: Настройка pre-commit hooks в Python проектах
date: 2024-05-02T15:30:00+00:00
category: Технические_Знания/Python
tags: [git, python, automation, ci-cd]
status: published
---
```

### ❌ Не делайте так:

```yaml
---
Title: настройка pre-commit (неправильный ключ, неправильное именование)
created_at: 02-05-2024 (неправильный формат даты)
cat: Tech/Python (неправильный ключ)
Tags: git, python (неправильный тип - должен быть список)
status: ready (неправильное значение)
---
```

## Интеграция с инструментами

### GitHub

Фронтматтер используется для:
- Фильтрации в GitHub Actions (будущее)
- Генерации метаданных в блоге (будущее)

### Поиск и фильтрация

Благодаря фронтматтеру, можно будет легко искать заметки:
```bash
# Будущее: поиск всех заметок по RAG
grep -r "tags.*rag" . --include="*.md"

# Поиск всех draft заметок
grep -r "status.*draft" . --include="*.md"
```

## Автоматическое обновление

При изменении заметки дата НЕ обновляется автоматически. Если хотите обновить дату:

1. Откройте файл
2. Измените `date` на текущее время в ISO 8601

Пример:
```yaml
date: 2024-05-03T14:30:00+00:00  # Обновленная дата
```
