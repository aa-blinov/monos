# Инструкции для разработчиков

## Первый запуск

После того как вы клонировали репозиторий, нужно настроить окружение.

### Требования

Перед стартом убедитесь, что установлены:
- **uv** — менеджер зависимостей Python (https://docs.astral.sh/uv/getting-started/)
- **git** — контроль версий
- **Python 3.8+**

### Инициализация (выбирайте по ОС)

**macOS / Linux:**
```bash
python3 setup.py
```

**Windows (PowerShell):**
```powershell
python setup.py
```

**Windows (Command Prompt):**
```cmd
python setup.py
# или
setup.bat
```

## Что делает скрипт инициализации?

1. ✅ Проверяет наличие `uv` и `git`
2. 📦 Устанавливает зависимости через `uv sync`
3. 🪝 Настраивает `pre-commit` hooks
4. 🎯 Подготавливает окружение к работе

## Работа с заметками (WebUI)

Все операции по управлению заметками выполняются через WebUI. 

1. **Запуск**: `cd webui && docker-compose up -d --build`
2. **Создание**: Кнопка "+" в интерфейсе.
3. **Форматирование**: Кнопка "Format" в Header.
4. **Синхронизация**: Кнопка "Sync" в Header.

## Git workflow

### Перед коммитом

Не волнуйтесь! Pre-commit hooks сделают всё сами:
- 📝 Отформатируют Markdown файлы (wrap 120 символов)
- 🐍 Проверят Python код через ruff
- ✓ Выполнят базовые проверки

Просто коммитьте как обычно:
```bash
git add .
git commit -m "Add new note about X"
```

### Если нужна ручная проверка

```bash
pre-commit run --all-files
```

### Пропуск проверок (не рекомендуется)

```bash
git commit --no-verify
```

## Структура проекта

Основные разделы:
- **Работа/** — Профессиональная деятельность (EORA, NSTU, Research DPI)
- **Проекты_и_Образование/** — Учебные проекты и самообучение
- **Технические_Знания/** — База знаний по технологиям (ЯМЗ, Python, AI)
- **Документы/** — Официальные документы (Digital Nomad, ВНЖ)
- **Личное_и_Творчество/** — Личное развитие и творчество
- **assets/** — Медиа файлы (images, docs)

## Правила именования

- Папки: используйте подчеркивания вместо пробелов (`Мой_Проект` не `Мой Проект`)
- Файлы заметок: `название.md` на русском или английском, как удобнее
- Коммиты: пишите на русском или английском, но ясно и кратко

## Если что-то пошло не так

### Pre-commit вызывает ошибки

Обычно это значит, что нужно какое-то форматирование. Скрипт часто исправляет ошибки сам, просто попробуйте коммитить снова.

### Зависимости не установились

Убедитесь, что `uv` установлен правильно:
```bash
uv --version
```

Если нужно переустановить окружение:
```bash
uv sync --reinstall
```

### На Windows скрипты не запускаются

Убедитесь, что PowerShell позволяет запускать скрипты:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```
