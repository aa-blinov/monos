## 1. Общие положения

**Название проекта:** MediaSquash

**Цель:** Создать десктопное приложение для интеллектуального сжатия видео и изображений с осознанными пресетами, скрывающими сложность FFmpeg за контекстными сценариями использования.

**Целевая аудитория:** Контент-мейкеры, разработчики, менеджеры — все, кто регулярно конвертирует видео и изображения без необходимости разбираться в кодеках и параметрах кодирования.

**Лицензия:** MIT (open source, без монетизации)

**Ограничения:** Использование эмодзи в интерфейсе и коде строго запрещено.

**Фреймворк UI:** Flet (кроссплатформенное решение на базе Flutter, простое в разработке и развертывании)

## 2. Функциональные требования

### 2.1 Базовые возможности

Приложение должно поддерживать следующий функционал:

- **Drag and Drop**: Пользователь может перетаскивать файлы непосредственно в окно приложения для добавления их в очередь обработки. Должна быть визуальная обратная связь при наведении файлов на зону приема.
- **File Picker**: Интеграция с системным диалогом выбора файлов для альтернативного способа загрузки видео и изображений. Поддержка множественного выбора.
- **Пакетная обработка**: Возможность добавлять несколько файлов в очередь и обрабатывать их последовательно без участия пользователя.
- **Очередь задач**: Визуальный просмотр всех задач (pending, processing, completed, failed) с возможностью отмены текущей задачи, повтора завершенной или удаления из списка.
- **Артефакты**: Каждая завершенная задача генерирует скачиваемый результат. Артефакт сохраняется в контейнере в директории /app/artifacts и доступен для скачивания на хост-машину через интерфейс приложения.
- **Поддержка видео**: Работа с форматами MP4, MOV, MKV, WebM, AVI, FLV и другими, поддерживаемыми FFmpeg. На вход приходит любой видеоформат, выход определяется пресетом и выбранным форматом.
- **Поддержка изображений**: Работа с форматами JPEG, PNG, WebP, AVIF, TIFF, BMP, GIF. Обработка через Pillow для оптимизации и конвертации.

### 2.2 Пресеты

Пресеты организованы по пяти категориям. Каждый пресет содержит:

- Понятное название на русском и английском
- Подробное описание сценария использования
- Полную отображаемую команду FFmpeg для прозрачности
- Целевые параметры выходного файла (разрешение, битрейт, кодек, и т.д.)

#### Категория: Social Media (7 пресетов)

Пресеты для популярных социальных сетей и мессенджеров:

1. **Instagram Reels**: Вертикальное видео формата 9:16 для Stories и Reels. Максимальная длительность 60 секунд, оптимизировано под алгоритмы сжатия Instagram. Использует H.264 с CRF 23 и битрейтом видео 3500 кбит/с.
1. **Instagram Feed**: Горизонтальное или квадратное видео для ленты. Адаптивное разрешение 1080x1080. H.264 кодирование с балансировкой качества.
1. **YouTube Upload**: Оптимальные настройки для загрузки на YouTube с минимальными потерями перед обработкой платформой. Использует H.264 high profile, более высокий CRF для сохранения качества.
1. **YouTube Shorts**: Вертикальный формат для YouTube Shorts. Специализированные параметры для вертикального видео с высоким битрейтом.
1. **TikTok**: Полная совместимость со спецификациями TikTok. Строгое соблюдение профиля H.264 baseline, поддержка мобильных устройств.
1. **Telegram Video**: Адаптировано для отправки в мессенджер Telegram с балансом качества и размера. Разрешение 1280px, умеренный битрейт.
1. **WhatsApp Status**: Минимальный размер для статусов и пересылки. Агрессивное сжатие с быстрым кодированием (preset veryfast), разрешение 720px.

#### Категория: Web (4 пресета)

Пресеты для встраивания и использования видео на веб-сайтах:

1. **Web Optimized**: Минимальный вес для встраивания на сайты. Использует H.264 с оптимизацией для быстрой загрузки, флаг moov atom в начале файла.
1. **WebM Modern**: Формат WebM с кодеком VP9 для современных браузеров. Лучше сжимает чем H.264, открытый стандарт.
1. **Web AV1**: Передовой кодек AV1 для максимального сжатия в вебе. Медленнее кодирует, но обеспечивает наименьший размер при качественной передаче.
1. **Background Video**: Бесшовное зацикленное видео для фонов веб-сайтов. Оптимизация для непрерывного воспроизведения с tune=stillimage.

#### Категория: Storage (3 пресета)

Пресеты для долгосрочного хранения и архивирования:

1. **Archive H.265**: Максимальное сжатие с использованием кодека H.265 для долгосрочного хранения. Медленное кодирование (preset slow) обеспечивает лучший результат.
1. **Archive Master**: Сохранение оригинального качества без перекодирования видео потока (копирование потока). Только аудио может быть переконвертирована. Использует контейнер MKV.
1. **Archive Lossless**: Кодирование без потерь для критичных материалов. Использует H.264 с CRF 0 и медленным кодированием.

#### Категория: Content Type (4 пресета)

Пресеты оптимизированные под тип создаваемого контента:

1. **Screen Recording**: Оптимизация для скринкастов и записей экрана с большими областями статики. Использует tune=stillimage для лучшего сжатия изменяющихся областей.
1. **Animation**: Сохранение четких границ и плоских цветов в анимации. Более высокий CRF, tune=animation для лучшей работы с графикой.
1. **Film Content**: Настройки для кинематографичного контента с камеры. tune=film для естественных изображений, более высокий CRF для сохранения деталей.
1. **Gaming**: Высокий битрейт для динамичного игрового контента с быстрыми переходами. Более строгие параметры кодирования для минимизации артефактов.

#### Категория: Images (8 пресетов)

Пресеты для обработки и оптимизации изображений:

1. **JPEG Quality**: Сбалансированное сжатие JPEG для веба. Качество 85 с оптимизацией и прогрессивной загрузкой.
1. **JPEG Small**: Минимальный размер JPEG с приемлемым качеством. Качество 70, сжатие оптимизировано под размер.
1. **PNG Optimize**: Оптимизация PNG без потерь. Использует встроенную оптимизацию Pillow.
1. **WebP Modern**: Современный формат WebP для веба. Качество 85, несжатый режим (lossy) для баланса.
1. **WebP Lossless**: Сжатие WebP без потерь для графики и скриншотов. Сохраняет максимальное качество.
1. **AVIF Maximum**: Максимальное сжатие с кодеком AVIF. Requires pillow-avif-plugin, качество 80, быстрое кодирование.
1. **GIF Convert**: Конвертация видео в GIF (максимум 15 секунд). Использует оптимизированную палитру для минимизации размера.
1. **AVIF Animation**: Анимированный AVIF как альтернатива GIF. Значительно меньший размер при лучшем качестве анимации.

### 2.3 Форматы вывода

Поддерживаются следующие форматы вывода:

- **MP4**: Универсальный контейнер mp4 с поддержкой видеокодеков H.264/H.265 и аудиокодека AAC. Совместим с 99% устройств.
- **WebM**: Контейнер WebM с видеокодеками VP9/AV1 и аудиокодеком Opus. Для современных браузеров.
- **MOV**: Контейнер MOV для профессиональной работы. Поддерживает H.264 и ProRes видеокодеки.
- **MKV**: Матрёшка контейнер MKV с поддержкой множественных аудиодорожек, субтитров и глав. Для архивирования.
- **GIF**: GIF89a формат для анимаций и мемов. Без звука, максимум 15 секунд.
- **WebP**: Статические и анимированные WebP изображения. Альтернатива JPEG и GIF.
- **AVIF**: Современный формат AVIF для максимального сжатия. Требует поддержки плагина pillow-avif.

## 3. Архитектура

### 3.1 Технологический стек

**Backend:**

- Python 3.12-slim как основной интерпретатор (строгое требование)
- ffmpeg-python для асинхронного взаимодействия с FFmpeg
- Pillow >= 10.0 для обработки изображений
- pillow-avif-plugin для поддержки AVIF формата
- pydantic >= 2.0 для валидации данных и конфигураций
- asyncio для асинхронной обработки в целом
- aiofiles для асинхронной работы с файловой системой

**Frontend:**

- Flet как основной UI фреймворк (кроссплатформенное решение на базе Flutter)
- Встроенная поддержка темизации через Flet Theme API
- Векторные иконки из Material Design Icons

**Контейнеризация:**

- Docker Desktop для полной контейнеризации приложения
- Multi-stage build для оптимизации размера контейнера
- Python 3.12-slim как базовый образ
- FFmpeg устанавливается в контейнер на этапе сборки через apt-get

**Дизайн:**

- Темная тема по умолчанию
- Минималистичный интерфейс в стиле CapCut
- Акцентные цвета: синий и фиолетовый градиенты
- Скругленные углы элементов (border radius)
- Плавные анимации переходов между состояниями
- Typography: системные шрифты (Inter, SF Pro на Linux/macOS)

### 3.3 Контейнеризация

Приложение полностью контейнеризовано через Docker. Dockerfile использует Python 3.12-slim как базовый образ и устанавливает FFmpeg на этапе сборки. Docker Compose конфигурация предоставляет удобный способ запуска контейнера с правильными volume маппингами для доступа к входным файлам и артефактам.

Контейнер экспортирует директорию /app/artifacts как volume, что позволяет файлам, обработанным приложением, быть доступными на хост-машине. Все зависимости Python устанавливаются в контейнер без привязки к хост-системе, обеспечивая полную независимость окружения.

## 4. Пользовательский интерфейс (Flet)

### 4.1 Концепция дизайна

Интерфейс построен в стиле современных приложений для редактирования видео (CapCut). Основные принципы дизайна:

- Темная тема с темными нейтральными фонами и высокий контраст для текста
- Акцентные цвета (фиолетово-синие) используются только для интерактивных элементов и статуса выполнения
- Скругленные углы (border radius 8-16 пикселей) для всех контейнеров и кнопок
- Размытые фоны (backdrop blur) для модальных диалогов и всплывающих окон
- Плавные анимации (300-400 миллисекунд с ease-out кривой) для переходов между состояниями
- Минимальное использование визуальных элементов, фокус на контенте
- Полное отсутствие эмодзи везде в интерфейсе

### 4.2 Структура экранов

**Главный экран** отображает зону для drag-and-drop в центр окна с поддержкой перетаскивания файлов. Выше находится панель категорий пресетов (Social Media, Web, Storage, Content Type, Images) как горизонтальные переключаемые кнопки. При выборе категории ниже отображается сетка карточек пресетов с названием, описанием и отображением команды FFmpeg.

При выборе пресета в нижней части экрана показывается информация о выбранном пресете, опции выбора формата вывода и кнопка добавления выбранных файлов в очередь.

**Экран очереди задач** отображает три отдельные вкладки или раздела: "В обработке" (максимум одна активная задача), "В очереди" (ожидающие задачи), "Завершено" (отработанные задачи). Для активной задачи показывается прогресс-бар с процентом выполнения и оценкой времени. Для каждой завершенной задачи отображается:

- Исходный файл с размером и длительностью
- Выходной файл с размером и процентом сокращения
- Кнопка скачивания артефакта
- Кнопка открытия папки с файлом
- Кнопка повтора обработки
- Кнопка удаления из истории

**Браузер пресетов** предоставляет подробный просмотр всех доступных пресетов с полной информацией о каждом, включая точную команду FFmpeg, параметры, и примеры использования. Позволяет пользователю понять, что делает каждый пресет.

### 4.3 Основные компоненты Flet

**Drop Zone**: Визуальная зона для drag-and-drop с изменением стиля при наведении. Отображает сообщение о поддерживаемых типах файлов.

**File Picker**: Интеграция системного диалога выбора файлов через FilePickerDialog Flet с фильтрами по расширениям.

**Preset Category Selector**: Горизонтальная полоса переключаемых кнопок для выбора категории пресетов. Активная категория выделена акцентным цветом.

**Preset Card**: Компактная карточка с названием пресета, описанием, и отображением части FFmpeg команды. На клик показывает полные детали.

**Queue Monitor**: Список задач с индикаторами статуса (pending/processing/completed/failed). Поддерживает прокрутку при большом количестве задач.

**Job Card**: Детальная карточка завершенной задачи с информацией об исходнике, результате, и действиями (скачать, повторить, удалить).

**Progress Indicator**: Анимированный прогресс-бар для активной задачи обработки с отображением процента и оценкой времени.

## 5. Асинхронная обработка

### 5.1 Модель обработки

Все операции, связанные с кодированием и обработкой файлов, выполняются асинхронно. GUI остается отзывчивым во время обработки видео и изображений благодаря использованию asyncio.

Очередь задач поддерживает добавление новых файлов во время обработки текущей задачи. Система обрабатывает по одной задаче последовательно, но архитектура позволяет легко масштабировать на параллельную обработку в будущем.

### 5.2 Жизненный цикл задачи

Каждая задача проходит следующие состояния:

1. **PENDING**: Задача добавлена в очередь и ожидает обработки
1. **PROCESSING**: Задача в настоящий момент обрабатывается, отправлены команды FFmpeg/Pillow
1. **COMPLETED**: Задача успешно завершена, артефакт доступен для скачивания
1. **FAILED**: Задача завершилась с ошибкой, отображается сообщение об ошибке
1. **CANCELLED**: Задача отменена пользователем (только для состояния PROCESSING)

### 5.3 Обновление прогресса

Для видео обрабатываемых FFmpeg парсится stderr поток для извлечения информации о прогрессе (текущий frame, duration, speed). Эта информация используется для обновления прогресс-бара в real-time.

Для изображений используется спиннер вместо прогресс-бара, так как обработка изображения обычно быстра.

## 6. Данные и конфигурация

### 6.1 Модели данных

Все модели данных определены через Pydantic для строгой валидации. Основные модели включают:

- **Preset**: Полное определение пресета с параметрами FFmpeg, описанием, и метаданными
- **Job**: Описание задачи обработки с информацией о входном/выходном файле, статусе, и прогрессе
- **JobStatus**: Перечисление состояний задачи
- **Category**: Перечисление категорий пресетов
- **ProcessingResult**: Результат обработки с информацией об исходном и выходном файле, времени обработки

### 6.2 Хранение пресетов

Пресеты определены в Python коде и поддерживаются в синхронизации вручную. Это обеспечивает полный контроль над параметрами и позволяет пользователю видеть точную команду FFmpeg для каждого пресета без необходимости парсинга конфигурационных файлов.

## 8. Требования к обработке файлов

### 8.1 Видеообработка

Для видео используется FFmpeg через subprocess с асинхронной обработкой. Команды генерируются на основе выбранного пресета и формата вывода. Поддерживаются все форматы, которые может прочитать FFmpeg.

Параметры кодирования включают выбор видеокодека (H.264, H.265, VP9, AV1), битрейта, CRF, скорости кодирования (preset), и видеофильтров (масштабирование, кроп, паддинг).

### 8.2 Обработка изображений

Для изображений используется Pillow с асинхронным запуском в thread pool executor. Обработка включает масштабирование при необходимости, оптимизацию, и сохранение в целевом формате.

Поддерживаются фильтры для конвертации между форматами (JPEG, PNG, WebP, AVIF) с параметрами качества и оптимизации, специфичными для каждого формата.

## 9. Требования к UI/UX на Flet

1. **Отсутствие эмодзи**: Весь интерфейс использует только текст и Material Design иконки, нет эмодзи нигде.
1. **CapCut Style**: Темная тема, фиолетово-синие акценты, скругленные углы на всех элементах, размытие фонов в диалогах.
1. **Drag and Drop**: Основной способ загрузки файлов, визуальная обратная связь при наведении (изменение цвета, размера зоны).
1. **File Picker**: Нативный диалог выбора файлов системы для альтернативного способа загрузки.
1. **Прозрачность**: Отображение полной команды FFmpeg или параметров обработки для каждого пресета.
1. **Очередь**: Визуальное отображение всех задач с возможностью отмены активной, повтора завершенной, удаления из истории.
1. **Артефакты**: Каждый завершенный файл сразу доступен для скачивания на хост через кнопку.
1. **Прогресс**: Детальный прогресс-бар с оценкой времени для видео, спиннер для изображений.
1. **Ошибки**: Человекочитаемые сообщения об ошибках с возможностью просмотра полного лога FFmpeg.
1. **Категоризация**: Пресеты четко разделены по категориям (Social Media, Web, Storage, Content Type, Images) в интерфейсе для легкого поиска.

## 11. Критерии приёмки

1. Пользователь без знания FFmpeg может обработать видео или изображение за 3 клика (выбрать файл, выбрать пресет, запустить).
1. Все пресеты работают корректно и производят результаты в соответствии с описанием.
1. Размер выходного файла соответствует заявленному в интерфейсе с точностью ±10%.
1. Приложение не падает при обработке битых или коротких файлов, выводит понятные сообщения об ошибках.
1. GUI остается отзывчивым во время обработки, очередь задач обновляется в real-time.
1. Артефакты сохраняются в контейнере и доступны для скачивания на хост-машину.
1. Контейнер собирается без ошибок и запускается на любом хосте с Docker Desktop.
1. Все пресеты отображают корректные команды FFmpeg в интерфейсе.
