#### 4. Аудит бэкенда

##### 4.1 Критичные риски (исправлены)

1. **Жёстко закодированные API-ключи**

   - **Проблема**: Действующие Google API Key и cloud embedding ключи в исходном коде
   - **Риск**: Доступ к внешним сервисам, исчерпание лимитов, атаки на инфраструктуру
   - **Решение**: Ключи вынесены в pydantic-настройки, хардкод удалён

1. **Небезопасная десериализация (`pickle`)**

   - **Проблема**: `ResponseCache` использует `pickle.loads()` из Redis
   - **Риск**: Выполнение произвольного кода при подмене ключа в Redis
   - **Решение**: Заменено на JSON-кодирование с валидацией структуры

1. **CSV Formula Injection**

   - **Проблема**: Пользовательский ввод в CSV без экранирования (экспорт подсказок)
   - **Риск**: При открытии в Excel формулы `=CMD(…)` могут привести к краже данных
   - **Решение**: Добавлена функция `sanitize_csv_field()` с экранированием опасных префиксов

##### 4.2 Итоговый статус бэкенда

- Критичные утечки секретов устранены
- Небезопасная сериализация исправлена
- CSV injection защищена
