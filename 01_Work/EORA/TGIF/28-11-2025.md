## 1. Что такое MCP и зачем он нужен

Model Context Protocol (MCP) представляет собой открытый стандарт для безопасного подключения AI-приложений к внешним системам. Протокол обеспечивает стандартизированный интерфейс для доступа к данным, инструментам и рабочим процессам.

### Преимущества MCP

**Разработчикам.** Сокращает время разработки AI-приложений в разы за счет стандартизации интерфейсов.

**AI-приложениям.** Предоставляет доступ к экосистеме источников данных, инструментов и приложений, улучшая функциональность и пользовательский опыт.

**Пользователям.** Позволяет AI-приложениям работать с персональными данными и выполнять действия от имени пользователя.

### Когда могут потребоваться MCP

MCP целесообразно использовать в следующих сценариях:

- **Интеграция с корпоративными системами.** Когда AI необходимо обращаться к внутренним базам данных, ERP-системам или API компании.
- **Работа с персональными данными.** Для доступа к календарям, email, документам пользователя с соблюдением политик безопасности.
- **Расширение функциональности AI.** Когда требуется интеграция с внешними инструментами: генераторами кода, 3D-моделлерами, аналитическими платформами.
- **Многосерверные архитектуры.** Для создания распределенных систем, где AI взаимодействует с несколькими специализированными серверами.
- **Безопасная изоляция.** Когда необходимо четко разделить доступ к данным и инструментам между разными пользователями или приложениями.

## 2. Как FastMCP помогает создавать серверы быстро

FastMCP представляет собой высокоуровневую Python-библиотеку для быстрой разработки MCP-серверов. Библиотека абстрагирует низкоуровневые детали протокола, позволяя сосредоточиться на бизнес-логике.

### Основные концепции FastMCP

**FastMCP сервер.**

```python
from fastmcp import FastMCP

mcp = FastMCP("Сервер данных")

@mcp.tool
def add(a: int, b: int) -> int:
    """Выполняет сложение двух чисел"""
    return a + b

if __name__ == "__main__":
    mcp.run()

```

**Что происходит в коде:**

- `FastMCP("Сервер данных")` - инициализация сервера
- `@mcp.tool` - декоратор для создания инструмента из функции
- `mcp.run()` - запуск сервера

### Транспорты: stdio, sse, http

FastMCP поддерживает три типа транспорта для различных сценариев развертывания.

#### STDIO (Standard Input/Output)

```python
mcp.run(transport="stdio")
```

**Когда использовать?** Локальные инструменты и CLI-приложения.

#### HTTP

```python
mcp.run(transport="http", host="127.0.0.1", port=8000, path="/mcp")
```

**Когда использовать?** Веб-сервисы и REST API интеграции.

#### SSE (Server-Sent Events)

```python
mcp.run(transport="sse", host="127.0.0.1", port=8000)
```

**Когда использовать?** Совместимость с существующими SSE-клиентами.

### Инструменты, ресурсы и промпты

MCP определяет три основных типа компонентов, которые AI может использовать для взаимодействия с внешними системами: инструменты, ресурсы и промпты.

**Инструменты (Tools).** Предназначены для выполнения действий и вызова функций. Инструменты позволяют AI выполнять операции, изменять данные или взаимодействовать с внешними системами.

**Когда LLM использует инструменты:**

- Когда нужно выполнить конкретное действие (отправить email, сохранить файл, вызвать API)
- Для обработки данных или выполнения расчетов
- При необходимости взаимодействия с внешними сервисами
- Когда требуется изменить состояние системы или выполнить операцию

**Когда модель сама вызывает инструменты:**

- Автоматически, когда запрос пользователя подразумевает выполнение действия
- На основе анализа описания инструмента и контекста запроса
- Когда инструмент лучше всего подходит для решения задачи
- При последовательном выполнении сложных операций (цепочка инструментов)

**Когда модель НЕ вызывает инструменты автоматически:**

- Если инструмент отключен или недоступенв
- Когда описание инструмента не соответствует задаче
- При недостатке информации для принятия решения
- Когда требуется явное подтверждение пользователя для критичных операций

**Важно!** Инструменты в MCP предназначены для автоматического управления моделью (model-controlled). Модель самостоятельно анализирует доступные инструменты и решает, когда их вызвать. Однако для обеспечения безопасности и доверия рекомендуется всегда иметь человека в цикле принятия решений с возможностью отклонить вызов инструмента.

```python

@mcp.tool(
    name="send_notification",  # Явное имя инструмента для модели
    description="Отправляет уведомление пользователю по email или SMS с указанным приоритетом и текстом сообщения"  # Подробное описание
)

def send_notification(user_id: str, message: str, priority: str = "normal") -> str:
    """Отправляет уведомление пользователю"""
    # Логика отправки
    return f"Уведомление отправлено пользователю {user_id}"

@mcp.tool
async def query_database(sql: str) -> dict:
    """Выполняет SQL-запрос к базе данных и возвращает результаты в формате JSON"""
    # Логика выполнения запроса
    return {"result": "запрос выполнен", "data": [...]}

```

**Важно!** Имя и описание инструмента критически важны для корректного выбора инструмента моделью. Модель анализирует описание, чтобы понять назначение инструмента и когда его использовать.

**Рекомендации по описаниям:**

- **Имя инструмента** должно быть понятным и уникальным
- **Описание** должно содержать:   - Что делает инструмент   - Какие параметры принимает   - Что возвращает   - Когда использовать   - Любые ограничения или особенности

**Примеры хороших описаний:**

```python

# Плохо - слишком общее
@mcp.tool(description="Обрабатывает данные")
def process_data(data: str) -> dict:
    pass

# Хорошо - конкретное и подробное
@mcp.tool(
    name="analyze_financial_data",
    description="Анализирует финансовые данные из CSV файла: рассчитывает доходы, расходы, прибыль, генерирует отчет с графиками и рекомендациями"

)
def analyze_financial_data(file_path: str, analysis_type: str = "full") -> dict:
    pass

```

**Ресурсы (Resources).** Предоставляют доступ к данным для чтения. Ресурсы позволяют AI получать информацию из различных источников без возможности изменения данных.

**Когда LLM использует ресурсы:**

- Для получения справочной информации (документы, конфигурации, метаданные)
- При необходимости прочитать содержимое файлов или баз данных
- Для доступа к статическим данным или настройкам
- Когда требуется контекстная информация для принятия решений

**Как ресурсы становятся доступными для модели:**

- **Автоматическое включение в контекст.** AI-модель может автоматически включать релевантные ресурсы в контекст на основе эвристики и анализа запроса пользователя
- **Выбор пользователем.** Пользователь может явно выбрать ресурсы через интерфейс клиента (например, через элементы управления UI, такие как resource picker)

**Когда модель автоматически включает ресурсы:**

- Автоматически при необходимости получения дополнительной информации
- Когда ресурс содержит релевантные данные для ответа
- Для проверки фактов или получения контекста
- При анализе или обработке данных

**Когда пользователь выбирает ресурсы:**

- При явном выборе из списка доступных ресурсов через UI
- Когда нужно предоставить специфический контекст для задачи
- Для доступа к конфиденциальным или ограниченным данным
- При работе с определенными файлами или наборами данных

**Когда ресурсы не используются:**

- Если ресурс недоступен или требует специальных разрешений
- Когда информация уже есть в контексте разговора
- При недостаточной релевантности ресурса к запросу
- Когда вызов ресурса может быть дорогостоящим или медленным

```python
@mcp.resource("config://version")
def get_version():
    return "1.0.0"

@mcp.resource("users://{user_id}/profile")
def get_user_profile(user_id: int):
    return {"name": f"User {user_id}", "status": "active"}
```

**Промпты (Prompts).** Определяют шаблоны для взаимодействия с AI. Промпты позволяют стандартизировать запросы к модели и предоставлять контекст для специфических задач.

**Когда LLM использует промпты:**

- Для структурирования сложных запросов с предопределенным контекстом
- При необходимости стандартизации формата взаимодействия
- Для предоставления модели дополнительных инструкций или примеров
- Когда требуется специализированная обработка определенного типа данных

**Важно!** Промпты в MCP предназначены для пользовательского контроля. Модель НЕ вызывает промпты автоматически - это делает пользователь через интерфейс клиента (например, через слэш-команды).

**Когда пользователь выбирает промпт:**

- При явном выборе из списка доступных промптов
- Когда нужно стандартизировать типовую задачу
- Для улучшения качества взаимодействия с моделью
- При работе с специфическими доменами (код, анализ данных, творчество)

**Когда промпты не используются:**

- При обычном диалоге без специальных шаблонов
- Когда пользователь хочет свободную форму общения
- При творческих или исследовательских задачах
- Когда стандартный промпт может ограничить креативность

```python
@mcp.prompt
def analyze_data_request(dataset: str) -> str:
    return f"Проанализируйте следующий набор данных и предоставьте ключевые insights:\n\n{dataset}"
```

### Контекст и дополнительные возможности

**Доступ к контексту сессии.**

```python
from fastmcp import Context

@mcp.tool
async def process_file(uri: str, ctx: Context):
    await ctx.info(f"Обработка файла {uri}")
    data = await ctx.read_resource(uri)
    analysis = await ctx.sample(f"Проанализируйте: {data.content[:500]}")
    return analysis.text
```

**Аутентификация.**

```python

mcp = FastMCP("Защищенный сервер", auth=auth)
```

## 3. Как написать простой клиент с Typer

Для тестирования и взаимодействия с MCP-серверами можно разработать клиентское приложение с использованием Typer.

### Базовый клиент

```python
import asyncio
import typer
from fastmcp import Client

app = typer.Typer()

@app.command()
def chat():
    """Интерактивный чат с MCP-сервером"""
    typer.echo("MCP Chat Client")

    async def run_session():
        async with Client("http://localhost:8000/sse") as client:
            tools = await client.list_tools()
            typer.echo(f"Доступные инструменты: {[t.name for t in tools]}")

            while True:
                user_input = typer.prompt("Запрос")

                if user_input.lower() == "/exit":
                    break

                try:
                    result = await client.call_tool("add", {"a": 5, "b": 3})
                    typer.echo(f"Результат: {result.content[0].text}")
                except Exception as e:
                    typer.echo(f"Ошибка: {e}")

    asyncio.run(run_session())

if __name__ == "__main__":
    app()
```

### Продвинутый клиент с поддержкой истории

```python
import asyncio
import typer
from fastmcp import Client

app = typer.Typer()

class ChatSession:
    def __init__(self):
        self.history = []

    async def interact_with_server(self, server_url: str):
        async with Client(server_url) as client:
            while True:
                user_input = typer.prompt("Запрос")

                if user_input == "/exit":
                    break

                elif user_input == "/tools":
                    tools = await client.list_tools()
                    typer.echo("Доступные инструменты:")
                    for tool in tools:
                        typer.echo(f"  - {tool.name}: {tool.description}")
                else:
                    self.history.append({"role": "user", "content": user_input})
                    # Обработка запроса через сервер
                    typer.echo("Обработка запроса...")

@app.command()
def chat(server_url: str = "http://localhost:8000/sse"):
    """Чат с MCP-сервером"""
    session = ChatSession()
    asyncio.run(session.interact_with_server(server_url))

if __name__ == "__main__":
    app()

```
