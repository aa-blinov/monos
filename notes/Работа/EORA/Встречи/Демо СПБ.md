Добрый день, коллеги. Меня зовут Саша, я технический лидер команды Eora. Представляю вам платформу Eora Dialog — решение для создания интеллектуальных диалоговых систем, доступное как готовый интерфейс или фреймворк для быстрой разработки чат-интерфейсов с широким спектром интеграций.

Сегодня я продемонстрирую панель администратора и рабочее пространство специалистов, которые ежедневно создают диалоговые системы для решения бизнес-задач. Для демонстрации используется очищенная версия платформы, но это не повлияет на понимание функционала и возможностей кастомизации.

**Архитектура платформы**

Платформа состоит из пяти основных разделов: дашборд, конструктор, чат, пользователи и классификаторы. Рассмотрим каждый из них.

**Дашборд — аналитика и метрики**

Дашборд является ключевым экраном для специалистов, занимающихся оптимизацией контакт-центра. Он предоставляет комплексный обзор метрик, критичных для управления операционной эффективностью.

**Важно отметить**: сейчас вы видите релевантные данные, но мы проводим кастомизацию под нужды клиентов.

**Конструктор — создание сценариев**

Конструктор предназначен для разработки диалоговых сценариев и управления связями между ними. Сценарий может представлять как полный диалог, так и его отдельный блок — элемент автоматизируемого взаимодействия.

Мы используем модульную парадигму построения сценариев. Это обеспечивает переиспользование компонентов между различными ботами и создание масштабируемой архитектуры. Ключевое преимущество — управляемость пользовательских флоу без риска запутаться в логике диалогов.

**Типы блоков сценария**

- **Точка входа** — инициализация сценария, чтобы было понятно, где начало
- **Условие** — ветвление логики на основе выполнения условий, в формате "если \<условие>, то \<ветка_1>, иначе \<ветка_2>
- **Ответ бота** — формирование ответа автоматизированной системы с подстановкой параметров
- **Интеграции** — интеграция с внешними системами (например, CRM): обращение по имени-отчеству, предоставление информации о счетах, перевод на оператора, оформление заявок
- **Ввод пользователя** — сбор и сохранение ответов клиентов в переменные для дальнейшей обработки
- **Классификатор** — интеллектуальная обработка множественных синонимов и смысловых вариантов. Например, "кредитная карта", "кредитка", направляются в единый диалоговый путь без необходимости прописывать все варианты вручную

Каждая интеграция возвращает переменные, доступные для повторного использования в других сценариях.

**Интерфейс тестирования**

В разделе "Чат" можно выбрать сценарии для тестирования, определить точку входа и проверить как изолированные блоки, так и полный сценарий целиком.

**Управление пользователями**

Система поддерживает две основные роли: администраторы с полным доступом и вьюеры с правами только на чтение (аналитика, чат, классификаторы без возможности редактирования). При необходимости можем расширить систему ролей — например, добавить модераторов для работы только с классификаторами или конструктором. В большинстве случаев базовых двух ролей достаточно для эффективного управления.

Супервизоры меняют сценарии (editor), операторы только читают (viewer).

## RAG — поиск по базе знаний

Eora RAG Platform заточена под создание индивидуальных ассистентов, которые решают задачи бизнеса по работе с базами знаний и отлично подходят для использования в контакт-центрах. Чтобы не превращать демонстрацию в затянутый рассказ, остановлюсь только на ключевых моментах, которые критичны для внедрения.

Процесс обработки запроса включает интеллектуальный поиск по базе знаний, проверку релевантности извлечённых документов, адаптацию под контекст (историю диалога и документы из БЗ) и формирование персонализированных ответов, которые настраиваются через инструкции.

Наше решение заняло 2-ое место на RRNCB, но при этом оно в несколько раз компактнее конкурентов при практически тех же метриках. Мы сознательно держимся стратегии баланса стоимости и качества.

**Архитектура базы знаний**

**Базы знаний** создаются на отдельном экране как изолированные модули, которые модерируются своими владельцами. Система использует продвинутый препроцессинг, который обрабатывает не только смыслы документов, но и работает с ключевыми словами, а также поддерживает широкий спектр форматов — от стандартных PDF и DOCX до прочих текстовых форматов.

Затем на экране **ассистентов** нужная база знаний подключается к конкретному ассистенту. Каждый ассистент консультирует по определённой предметной области, автоматизирует работу с конкретным фрагментом базы знаний и может быть персонализирован через специальные инструкции
